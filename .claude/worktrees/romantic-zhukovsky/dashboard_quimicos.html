Atualização Automática — Guia de Configuração
Escolha a melhor estratégia para sincronizar com o OneDrive / SharePoint
1
Python Script — Watchdog + Flask API
Recomendado
Monitor local ou em servidor que detecta mudanças no arquivo Excel e dispara atualização via API REST. Ideal para uso em rede local ou servidor dedicado.
▸Instalar: pip install watchdog openpyxl flask flask-cors
▸Script watcher.py monitora o arquivo com watchdog.Observer — detecta on_modified
▸Ao detectar mudança: lê o XLSX com openpyxl, converte para JSON e salva em /api/precos
▸Dashboard faz fetch('/api/precos') a cada intervalo ou via WebSocket para dados em tempo real
▸Agendar via Agendador de Tarefas (Windows) ou cron (Linux) para iniciar com o sistema
2
Power Automate — Trigger OneDrive → Webhook
Sem Código
Fluxo nativo no Microsoft 365 que detecta modificação no arquivo OneDrive/SharePoint e dispara webhook HTTP para o dashboard. Zero servidor necessário.
▸Criar fluxo em make.powerautomate.com → gatilho: "Quando um arquivo é modificado (OneDrive)"
▸Selecionar o arquivo historico_precos_quimicos.xlsx no caminho do SharePoint
▸Ação → "HTTP": POST para URL do dashboard ou Azure Function com os dados do Excel (via Get file content)
▸Alternativa: usar conector Excel Online (Business) para extrair tabela diretamente na ação
▸Dashboard recebe sinal e chama refreshData() — atualize com EventSource ou polling curto (30s)
3
Refresh Agendado — Microsoft Graph API + Cron
Totalmente Automático
Aplicação registrada no Azure AD que usa a Microsoft Graph API para ler o XLSX diretamente do SharePoint em intervalos regulares. Escalável e seguro.
▸Registrar App no portal.azure.com → Azure Active Directory → App Registrations
▸Permissões: Files.Read ou Sites.Read.All (Microsoft Graph)
▸Endpoint: GET /sites/{site-id}/drive/items/{item-id}/content → retorna o XLSX como stream
▸Processar com openpyxl ou pandas → gerar JSON → salvar em cache/banco
▸Cron job: */15 * * * * python fetch_graph.py (a cada 15 min) ou usar Azure Functions Timer Trigger
Dica para sua estrutura atual (SharePoint/OneDrive): A combinação ideal é Opção 2 (Power Automate) como trigger + Opção 3 (Graph API) para leitura — o Power Automate detecta a mudança e chama uma Azure Function que lê o arquivo via Graph API e atualiza um endpoint que o dashboard consome. Custo zero com licença M365 Business.
